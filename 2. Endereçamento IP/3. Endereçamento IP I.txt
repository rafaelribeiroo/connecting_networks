IP basicamente é um endereço que é utilizado para identificar os dispositivos
na rede.
    ← Em uma rede o cliente e também o servidor obrigatoriamente recebem um IP;
    ← Normalmente os endereços de servidores são fixos (estático) para que todo
mundo possa "achar" ele, pois quando trocam IP dele, geralmente tem que
replicar para toda a internet, demorando cerca de 24hrs;
    ← Geralmente os endereços residenciais variam (dinâmico), pois é muito
difícil a pessoa ficar conectada 24hrs por dia/7 dias por semana.

NOTA: IPv4 nós vemos em decimal e em IPv6 em hexadecimal, já nos dispositivos é
tudo binário.
    ← Um IPv4 é composto de quatro octetos;
    Por que são chamados octetos?
    ← Porque ao converter qualquer "octeto" para binário, teremos 8 bits.

Todos os IP's fazem parte de alguma classe – divisão em grupos, sendo eles:

CLASSE          IP                    ICANN
     A       0   a 127        2²⁴: 16.777.216 possibilidades
     B       128 a 191        2¹⁶: 65.536          //
     C       192 a 223         2⁸: 256             //
     D       224 a 239
     E       240 a 255

"O que é esse multicast citado na linha 29?"
    A comunicação pelos IP's podem ocorrer de quatro formas, sendo elas:
    ← Unicast: Envia para um único host (1 ↔ 1);
    ← Multicast: Envia para um grupo seleto de pessoas (1 ↔ GRUPO);
    ← Broadcast: Envia para todos dentro da rede. Por exemplo: BBC – o b de
broadcast (1 ↔ TODOS DENTRO DA REDE);
    ← Anycast: Envia para qualquer um próximo. Por exemplo: CDN (1 ↔ QUALQUER
UM PRÓXIMO).

Todo endereço IP é dividido em duas partes basicamente, dentre elas:
    ← endereço de rede: Determina a qual rede o IP pertence;
    ← endereço de host: Determina o dispositivo dentro da rede.
