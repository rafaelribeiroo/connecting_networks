Todos os IP's fazem parte de alguma classe (divisão em grupinhos):
CLASSE      IP               ICANN
A           0   a 127        2²⁴ → 16.777.216 possibilidades
B           128 a 191        2¹⁶ → 65.536
C           192 a 223        2⁸ → 256
D           224 a 239
E           240 a 255

Explicações:

As únicas classes que podem ser atribuídas a computadores na internet são
A // B // C

A classe D é reservada a multicast;

A classe E para teste de novas tecnologias (6g/7g)



A comunicação pelos IP pode ser de 4 formas
Unicast: Envia para um único host (1 pra 1) (dispositivo é algo que posso endereçar na rede)
Multicast (1 para um grupo), se voce tem gatonet voce recebe aquela mensagem (entre em contato com nosso suporte para que seu cpf nao seja), a net envia essa msg a todos que nao pagaram suas faturas
Broadcast (1 pra todos dentro dessa mesma rede) bbc, o primeiro b é de broadcast, eles enviam pra todos
Anycast (1 - qualquer um mais proximo), isso funciona mais com DNS.  Quando trabalhamos com IPv6, é possivel que temos varios servidores DNS com o mesmo endereço de IP em diversos locais, entao a requisição é feita pra qualquer DNS que esteja mais perto de vc

CONVERTENDO MASCARAS PADRAO PARA ABREVIADAS

mascara de rede padrão - cidr

a: 255.0.0.0.0 >> CORRESPONDENTE CIDR >> /8
b: 255.255.0.0 >> CORRESPONDENTE CIDR >> /16
c: 255.255.255.0 >> CORRESPONDENTE CIDR >> /24

De onde surgiu 8, 16 e 24? Porque quando convertemos de decimal pra binario temos:
quantos uns temos?

1 2 4 8 16 32 64 128 = 255
1 1 1 1  1  1  1   1 = 8bits 1

usando cidr nossa mascara fica um pouco diferente

255.255.255.192
 8   8   8

e o 192?
precisamos da tabelinha

sempre comece pelo maior valor e vai de tras pra frente
1 2 4 8 16 32 64 128
               1   1

entao essa mascara de rede padrão 255.255.255.192 corresponde ao CIDR /26
                                    8 + 8 + 8 + 2 = 26


VAMOS AO INVERSO AGORA, DE CIDR PARA PADRÃO
