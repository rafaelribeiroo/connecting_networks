Quando se tem uma rede de computadores, essencialmente temos dois grandes componentes de máquina.
	
	Servidor → Temos as máquinas que usam serviços e as máquinas que disponibilizam serviços. 

Downsizing: Tirar o mainframe dos terminais e transformar em vários computadores menores. Ao invés de ter um computador só, tenho vários com funções diferentes na rede.
	Foi aí que surgiu o cliente-servidor, nós que acessamos; E o servidor, que dispõe.

	Quando você acessa seu celular (ele é um cliente), aí você digita www.youtube.com, aí o DNS vai pegar esse hostname, encaminhar pro seu roteador e perguntar se ele sabe chegar, o roteador vai gerar um IP e vai encaminhar lá pro youtube (para os servidores da Google).

Servidor basicamente provê acesso a um serviço, disponibiliza recursos e serviços pra rede ou pra algum computador que esteja conectado a rede. Abaixo descrevemos alguns tipos de servidores.

Servidor de aplicação → Ele vai prover aplicações.
Servidor web → É como um servidor de arquivos, o que muda é a essência do tipo de arquivo (no de ARQUIVOS você pode incluir músicas, filmes, zips etc), já no web você vai colocar arquivos web (html, css, js, img).
Servidor BD → Guardar dados, fornecer os dados que estão pedindo.
